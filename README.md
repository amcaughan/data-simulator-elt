# data-simulator-elt

ELT pipeline for datasets generated by `data-simulator-api`.

This repository contains:
- containerized ingestion and standardization jobs
- a dbt project for Athena-based transforms
- Terragrunt/Terraform infrastructure for deploying the pipeline
- local development tooling
- CI for static analysis and security scanning

`aws_infra` manages the shared AWS account baseline and network.
`data-simulator-api` manages the private simulator API.
This repository manages ingestion, storage layout, and transforms.

Repository structure

- `jobs/` for ingestion and standardization code
- `dbt/` for the dbt project and model layout
- `infra/terragrunt/modules/` for Terraform modules local to this repo
- `infra/terragrunt/live/` for deployable stacks
- `docker/dev/` for the local development container

Planned pipeline shape

- ECS task for API ingestion inside the shared dev VPC
- landing, raw, and analytics zones in S3
- Athena and Glue metadata over standardized Parquet data
- dbt-athena models from raw to curated marts

Usage

Infrastructure is applied manually via Terragrunt.
CI should remain limited to static analysis and security scanning unless there is a deliberate reason to automate deploys.
